# In-app update check + install (v0.8.0)

```
HANDOFF ATOMIC STEP: v0.8 R0–R5 — check + user-initiated portable apply
PAUSED / CANCELLED:    Silent auto-replace; MSI/MSIX; non-GitHub mirrors; requiring Authenticode to apply
CANONICAL OWNER:       guardian-core::update + guardian-service + apps/unstick-updater
PROOF BEFORE DONE:     L1 parse/semver/hash; L2 check + apply on copy of dist\; hash mismatch aborts
```

## Goal

Discover a newer **GitHub Latest** for `Dendro-X0/Unstick` and, on explicit user confirm, download / verify / stop / extract-over-install / restart — same portable story as USER-GUIDE, without leaving the app. Keep `%LOCALAPPDATA%\Unstick\` untouched.

## Locked decisions

| Item | Choice |
|------|--------|
| Channel | `https://api.github.com/repos/Dendro-X0/Unstick/releases/latest` |
| Asset | `Unstick-{semver}-windows-x64.zip` only |
| Digest | Companion asset `SHA256SUMS` (or `{zip}.sha256`) generated by `Package-Portable.ps1` |
| Trigger | Background **check** optional; **Install** always user-confirmed |
| Replace | Sibling `unstick-updater.exe` (cannot overwrite running self) |
| Install dir | Parent of `guardian-service` `current_exe()` |
| Staging | `%LOCALAPPDATA%\Unstick\updates\` |
| Unsigned | Allowed with honesty banner; hash still required |

## IPC / status

### `ClientRequest`

- `CheckForUpdate` — fetch Latest; update status fields; return `Ok` / `Error`
- `StartUpdate` — download + verify + spawn updater + schedule process exit (only when update available and idle enough)

### `StatusSnapshot` fields

| Field | Meaning |
|-------|---------|
| `update_check_enabled` | From config (default true) |
| `update_available` | True when Latest semver > running `VERSION` |
| `update_version` | Tag without leading `v` (e.g. `0.8.1`) |
| `update_notes_url` | `html_url` of the release |
| `update_state` | `idle` \| `checking` \| `available` \| `downloading` \| `verified` \| `applying` \| `error` |
| `update_error` | Last human-readable failure (empty when ok) |
| `update_unsigned_warning` | True when channel is unsigned / SIGNING honesty applies |

### Config

- `update_check_enabled` (default `true`)
- `update_check_interval_secs` (default `86400` — daily background check when service runs)
- `update_snooze_until` optional RFC3339 — skip nag until then (optional; may ship as no-op in 0.8 if unused by UI)

## Flow

1. **Check** — GET Latest JSON → parse `tag_name`, assets → find zip + SHA256SUMS → compare semver to `guardian_core::VERSION`.
2. **Install** — Download zip + sums to `updates\` → verify SHA256 of zip bytes → soft-restore Live demotions → spawn `unstick-updater.exe --install-dir … --zip … --restart-service --restart-ui` (and `--restart-tray` if tray was running) → exit service (UI/tray exit via updater kill or self-exit).
3. **Updater** — Wait briefly for parents to exit → `Stop-Process` names `guardian-service`, `guardian-ui`, `guardian-tray` → extract **allowlisted** members only into install dir → start service then UI/tray as requested → exit 0.

### Extract allowlist

- `guardian-service.exe`, `guardian-ui.exe`, `guardian-tray.exe`, `unstick-updater.exe`
- `Install-Autostart.ps1`, `Uninstall-Autostart.ps1`
- `USER-GUIDE.md`, `RELEASE-NOTES.md`, `SIGNING.txt`, `README.txt`, `packaging-and-soak.md`, `frontend-spec.md`

Reject zip entries with `..`, absolute paths, or names outside the allowlist.

## Failure modes

| Failure | Behavior |
|---------|----------|
| Network / GitHub error | `update_state=error`, message; no binary change |
| No matching asset | Error; no apply |
| SHA256 mismatch / missing digest | Abort; delete bad download; no extract |
| Updater missing beside service | Error; ask manual zip update |
| Extract partial | Prefer write-to-temp-then-rename per file; on failure leave previous EXEs and set error log under `updates\last-apply.log` |

## Trust (unsigned era)

1. HTTPS + fixed owner/repo  
2. Asset name pattern pin  
3. SHA256 of zip must match published sums  
4. After extract, if Authenticode present on new EXEs, note in log; if unsigned, UI already warned at confirm  

## Out of scope (0.8)

Silent auto-replace; delta patches; user-picked zip paths; MSI; non-GitHub channels.

## Proof

- **L1:** semver compare; parse release JSON; parse SHA256SUMS; reject path traversal names  
- **L2:** Check against live Latest (or fixture); Install on a **copy** of `dist\`; wrong hash refuses  
- **Ship:** zip includes `unstick-updater.exe` + `SHA256SUMS` asset next to versioned zip
