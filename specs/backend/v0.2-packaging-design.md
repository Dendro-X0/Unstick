## Decision (locked for v0.2)

**Primary path:** signed **portable zip** on GitHub Releases (same layout as v0.1.x), not MSI/MSIX in the first v0.2 tag.

| Choice | Rationale |
|--------|-----------|
| Portable zip first | Already proven (`Package-Portable.ps1`); zero Store/MSI toolchain debt |
| Code signing required for ‚Äúpublic‚Äù (V2-3) | SmartScreen trust; unsigned stays private-beta banner |
| MSI/MSIX | Deferred to v0.2.x or v0.3 once Authenticode + update channel work |

## Update story (user-facing)

1. Download `Unstick-X.Y.Z-windows-x64.zip` from the GitHub Release marked Latest.  
2. Stop Guard: close UI; stop `guardian-service` (Task Manager or `Uninstall-Autostart.ps1` without deleting config).  
3. Extract zip **over** the existing install directory (replace exes; keep `%LOCALAPPDATA%\Unstick\` config).  
4. Start `guardian-service.exe`, then `guardian-ui.exe` (or re-run `Install-Autostart.ps1 -StartNow`).  
5. Confirm version chip / status `version` matches the tag.

No auto-updater in v0.2. Manual replace is the supported channel.

**v0.8+:** In-app check + user-initiated install supersedes ‚Äúno updater‚Äù for daily use ‚Äî see [in-app-update-design.md](in-app-update-design.md). Manual zip extract remains the documented fallback.

## Signing (ops)

| Step | Owner |
|------|-------|
| Obtain Authenticode cert (or cloud signing) | Maintainer |
| `signtool sign /fd SHA256 ‚Ä¶` on `guardian-service.exe`, `guardian-ui.exe`, `guardian-tray.exe` before zip | Release script extension |
| Publish only signed zip as public Latest; unsigned = prerelease / private | Release notes honesty |

Until a cert exists: keep releases labeled **unsigned portable** (current practice); V2-3 blocks calling the SKU ‚Äúpublic production.‚Äù

## Package script delta (implement when signing ready)

- Read version from workspace `Cargo.toml` (already done in v0.1.2).  
- **`-Sign` switch** invokes `signtool` when `UNSTICK_SIGN_THUMBPRINT` is set (or `/a` auto).  
- Fail public release when `REQUIRE_SIGN=1` and signing fails.  
- `SIGNING.txt` in the zip states signed vs unsigned honestly.
